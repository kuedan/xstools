Introduction

This script was created by It’sMe to help Xonotic server admins to manage several Xonotic git, Xonotic release servers and rcon2irc bots. Those servers and bots can easily called by their name and will be loaded into seperate tmux windows. Next to those functions to manage servers and bots, this tool offers a convenient folder tree, which simplifies managing configuration and other files like .pk3 packages. All adjustments can be made in the basic configuration file called xstools.conf.

  • required Software:

    • tmux

  • optional Software:

    • perl (for rcon2irc bots),

    • git (for Xonotic git servers),

    • qstat (to display different server informations)

Installation

First you have to download Xonotic Server Tools. You can download the last packed zip file or the current development zipball. The development zipball may not work as expected. The unzipped zip file will be your new Xonotic user directory (default is ~/.xonotic). You can move the and name the folder as you like.

Like mentioned you can control two different server versions: Release and git servers. The easiest way is to run release servers. Those are stable and recommended. Before you install any of those versions you have to set some configuration issues inside of xstools.conf, which can be found in configs/.

Please set the following variables:

userdir                      - location of your Xonotic userdir

basedir_release              - basedir of your release installation
basedir_git                  - basedir of your git installation
default_version              - default used version, release or git
Note
 If you are going to host release servers, you need to set basedir_release, otherwise set basedir_git. You can also host both server versions, therefore set both paths.
Note
 In every case you have to set a default_version.

To download the last official release, visit xonotic.org. Download the .zip file and extract it into your choosed basedir_release folder.

To host git servers you have to install git, a compiler and some libraries (check dev.xonotic.org). After installing those dependencies you can clone and compile the git repository via ./xstools.sh --install-git (just follow the instructions). To make xstools.sh easier accessible, move xstools (not xstools.sh) to a location, that is part of your search path for commands (PATH).

      ├── configs
      │   ├── rcon2irc         - your rcon2irc configuration files
      │   └── servers          - your individual server configuration files
      │       └── common       - your 'common' server configuration files
      ├── data                 - the default 'data' folder
      │   ├── logs             - all generated logs will be here
      │   └── maps             - the default map folder - add your mapinfo files here
      ├── logs -> data/logs/   - just a symlink
      └── packages             - place your packages (.pk3) here
